# Roadmap

## Near Term

- Stabilize the first public open-source release
- Add richer shared contracts between Driver and Brain
- Improve cross-platform packaging and release automation
- Tighten contributor onboarding for isolated Desktop, Driver, and Brain workflows

## Mid Term

- Expand protocol documentation and compatibility testing
- Introduce stronger end-to-end coverage for Desktop to Driver to Brain flows
- Improve plugin or extension points for tools and skills
- Harden security review around tool execution and file-edit workflows

## Longer Term

- Improve portability of release builds across macOS, Windows, and Linux
- Reduce duplication between runtime protocol definitions and docs
- Explore optional remote or team workflows without losing the local-first architecture
