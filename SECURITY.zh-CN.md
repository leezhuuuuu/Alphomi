# 安全策略

[English](SECURITY.md) | 简体中文

## 关注范围

以下方向的安全报告对 Alphomi 尤其有价值：

- Desktop、Driver 与 Brain 之间的本地进程编排
- 浏览器自动化边界
- prompt 注入或工具注入路径
- 文件系统访问与补丁应用流程
- 配置、密钥以及发布打包相关问题

## 如何报告漏洞

请不要为疑似安全漏洞创建公开 issue。

推荐流程：

1. 私下联系维护者，或在启用时使用 GitHub Security Advisories
2. 提供清晰描述、受影响文件或流程、复现步骤以及潜在影响
3. 如果可能，请附最小复现样例和建议的缓解方式

## 建议包含的信息

- 受影响版本或 commit
- 攻击前提条件
- 触发方式或载荷
- 预期影响
- 已知临时规避方式

## 响应预期

项目会尽量做到：

- 及时确认收到报告
- 复现并完成初步分级
- 说明严重程度与后续步骤
- 在修复或缓解方案可用后协调披露

## 密钥与本地测试

- 不要提交真实 API key 或 token
- 在本地使用 `config.yaml` 或环境变量管理敏感信息，不要把密钥写进 issue 或 PR
- 如果漏洞复现依赖私有凭据，请在分享前打码或替换
